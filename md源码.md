# Markdown 终极指南：从入门到精通

Markdown 是一种轻量级标记语言，它允许人们使用易读易写的纯文本格式编写文档。由于其简洁和高效，它已经成为程序员、写作者和文档工程师的首选工具。

## 1\. 标题 (Headings)

Markdown 使用井号 `#` 来表示标题，井号的数量对应标题的层级（H1 到 H6）。

### 演示：

**源码：**

```markdown
# 这是一级标题 (H1)
## 这是二级标题 (H2)
### 这是三级标题 (H3)
#### 这是四级标题 (H4)
```

**预览效果：**
(为了不破坏本文结构，此处仅展示 H3 和 H4)

### 这是三级标题 (H3)

#### 这是四级标题 (H4)

-----

## 2\. 文本样式 (Text Styling)

您可以轻松地对文本进行加粗、斜体或删除线处理。

### 演示：

**源码：**

```markdown
*这是斜体文本* (或 _这也是斜体_)
**这是加粗文本** (或 __这也是加粗__)
***这是加粗斜体文本***
~~这是删除线文本~~
```

**预览效果：**

  * *这是斜体文本*
  * **这是加粗文本**
  * ***这是加粗斜体文本***
  * \~\~这是删除线文本\~\~

-----

## 3\. 列表 (Lists)

列表分为无序列表、有序列表和任务列表。

### 3.1 无序列表

使用 `-`, `+`, 或 `*` 作为标记。

**源码：**

```markdown
- 项目 A
- 项目 B
  - 子项目 B.1 (前面加两个空格)
  - 子项目 B.2
* 项目 C
```

**预览效果：**

  * 项目 A
  * 项目 B
      * 子项目 B.1
      * 子项目 B.2
  * 项目 C

### 3.2 有序列表

使用数字加点 `1.`。数字本身不重要，Markdown 会自动排序，但建议从 1 开始。

**源码：**

```markdown
1. 第一步
2. 第二步
3. 第三步
   1. 第三步的细节
```

**预览效果：**

1.  第一步
2.  第二步
3.  第三步
    1.  第三步的细节

### 3.3 任务列表 (Task Lists)

常用于待办事项，使用 `[ ]` 和 `[x]`。

**源码：**

```markdown
- [x] 已完成的任务
- [ ] 待完成的任务
- [ ] 另一个待办
```

**预览效果：**

  - [x] 已完成的任务
  - [ ] 待完成的任务
  - [ ] 另一个待办

## 4\. 引用 (Blockquotes)

当您需要引用一段文字或进行强调说明时，使用 `>` 符号。

### 演示：

**源码：**

```markdown
> 这是一个引用段落。
>
> > 这是嵌套引用。
>
> 回到第一层引用。
```

**预览效果：**

> 这是一个引用段落。
>
> > 这是嵌套引用。
>
> 回到第一层引用。

## 5\. 代码 (Code)

这是开发者最喜欢的功能。

### 5.1 行内代码

使用单反引号 `` ` `` 包裹。

**源码：**

```markdown
请使用 `print("Hello World")` 函数。
```

**预览效果：**
请使用 `print("Hello World")` 函数。

### 5.2 代码块

使用三个反引号 ` ``` ` 包裹，并可以指定语言以实现语法高亮。

**源码：**

````markdown
```python
def hello():
    print("Hello, Markdown!")
```
````

**预览效果：**

```python
def hello():
    print("Hello, Markdown!")
```

## 6\. 链接与图片 (Links & Images)

两者语法非常相似，区别在于图片前面多一个感叹号 `!`。

### 6.1 超链接

**源码：**

```markdown
[点击访问 Google](https://www.google.com)
```

**预览效果：**
[点击访问 Google](https://www.google.com)

### 6.2 图片

**源码：**

```markdown
![替代文本](图片链接地址 "可选的标题")
```

## 7\. 表格 (Tables)

使用 `|` 分隔列，使用 `-` 分隔表头。可以通过冒号 `:` 控制对齐方式。

### 演示：

**源码：**

```markdown
| 左对齐 | 居中对齐 | 右对齐 |
| :--- | :----: | ---: |
| 单元格 | 单元格 | 单元格 |
| 内容 A | 内容 B | 内容 C |
```

**预览效果：**

| 左对齐 | 居中对齐 | 右对齐 |
| :--- | :----: | ---: |
| 单元格 | 单元格 | 单元格 |
| 内容 A | 内容 B | 内容 C |

## 8\. 分割线 (Horizontal Rules)

使用三个或更多的 `-`、`*` 或 `_` 来创建分割线。

**源码：**

```markdown
---
```

**预览效果：**

---
---
　　

## 9\. 高级功能 (Advanced)

### 9.1 数学公式 (LaTeX)

Markdown 支持使用 LaTeX 语法编写数学公式。

  * 行内公式使用 `$ ... $`
  * 块级公式使用 `$$...$$`

**源码：**

```markdown
质能方程是 $E=mc^2$。

复杂公式：
$$
f(x) = \int_{-\infty}^\infty \hat f(\xi)\,e^{2\pi i \xi x} \,d\xi
$$
```

**预览效果：**
质能方程是 $E=mc^2$。

复杂公式：

$$
f(x) = \int_{-\infty}^\infty \hat f(\xi)\,e^{2\pi i \xi x} \,d\xi
$$

### 9.2 转义字符

如果您想显示 Markdown 的语法符号（如 `*` 或 `#`）而不是应用其格式，请在符号前加反斜杠 `\`。

**源码：**

```markdown
\*这不是斜体\*
```

**预览效果：**
\*这不是斜体\*

## 总结

Markdown 的核心在于\*\*“所写即所得”\*\*的流畅感。您不需要鼠标点击复杂的菜单，只需专注于键盘输入。

> **小贴士：** 不同的平台（如 GitHub, Notion, Obsidian）对 Markdown 的支持可能会略有扩展（例如支持流程图 Mermaid 或脚注），但上述内容涵盖了 95% 的通用标准语法。
